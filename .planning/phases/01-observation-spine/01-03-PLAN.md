<objective>
Implement the operator config loader: YAML file at `/etc/mitigador/config.yaml` (path overridable via `--config`), with env-var overrides for every secret, validated at boot time so an invalid config fails fast with a clear message. Downstream plans (04: user CLI, 05: ingest, 09: alerts, 10: api) call `config.Load(path)` once and receive a fully-validated struct.

Purpose: Centralize all config concerns in one package. Every subsystem reads from a typed struct, not from `os.Getenv` or scattered viper calls. Validation runs once at boot — operator never reaches "service started but Telegram won't fire because the bot token is `CHANGE_ME`".

- Typed `Config` struct mirroring `deploy/examples/config.yaml`.
- `Load(path string)` reads YAML + applies env overrides.
- `Validate(*Config)` returns a multi-error per `go-playground/validator/v10`.
- Unit tests with `testdata/*.yaml` fixtures.

    2. Create `internal/config/load.go`:
       ```go
       package config

       import (
           "fmt"
           "os"
           "strings"

           "github.com/spf13/viper"
       )

       // DefaultPath is the canonical operator config location.
       const DefaultPath = "/etc/mitigador/config.yaml"

       // Load reads the YAML at path, applies MITIGADOR_<SECTION>_<KEY> env overrides,
       // and validates the result. Returns a typed Config or an error.
       func Load(path string) (*Config, error) {
           if path == "" {
               path = DefaultPath
           }
           if _, err := os.Stat(path); err != nil {
               return nil, fmt.Errorf("config: open %s: %w", path, err)
           }
           v := viper.New()
           v.SetConfigFile(path)
           v.SetConfigType("yaml")

           // Defaults.
           v.SetDefault("postgres.max_conns", int32(16))
           v.SetDefault("postgres.min_conns", int32(2))
           v.SetDefault("ingest.receive_buffer_bytes", 33554432)
           v.SetDefault("log.level", "info")
           v.SetDefault("log.format", "json")

           // Env overrides: MITIGADOR_POSTGRES_DSN, MITIGADOR_TELEGRAM_BOT_TOKEN, etc.
           v.SetEnvPrefix("MITIGADOR")
           v.SetEnvKeyReplacer(strings.NewReplacer(".", "_"))
           v.AutomaticEnv()

           if err := v.ReadInConfig(); err != nil {
               return nil, fmt.Errorf("config: read %s: %w", path, err)
           }

           var cfg Config
           if err := v.Unmarshal(&cfg); err != nil {
               return nil, fmt.Errorf("config: unmarshal: %w", err)
           }
           if err := Validate(&cfg); err != nil {
               return nil, err
           }
           return &cfg, nil
       }
       ```

       ```go
       package config

       import (
           "errors"
           "fmt"
           "strings"

           "github.com/go-playground/validator/v10"
       )

       // Validate runs struct-tag validation against the loaded config.
       // Returns a multi-error joining all failing fields, formatted human-readably.
       func Validate(cfg *Config) error {
           if cfg == nil {
               return errors.New("config: nil")
           }
           v := validator.New(validator.WithRequiredStructEnabled())
           if err := v.Struct(cfg); err != nil {
               var verrs validator.ValidationErrors
               if errors.As(err, &verrs) {
                   var sb strings.Builder
                   sb.WriteString("config: validation failed:")
                   for _, fe := range verrs {
                       fmt.Fprintf(&sb, "\n  - %s: failed %q (got %q)", fe.Namespace(), fe.Tag(), fmt.Sprint(fe.Value()))
                   }
                   return errors.New(sb.String())
               }
               return fmt.Errorf("config: validate: %w", err)
           }
           return nil
       }
       ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Filesystem → Config struct | YAML file at `/etc/mitigador/config.yaml` carries secrets (DB password, bot token, SMTP password, session secret) |
| Env vars → Config struct | Env overrides take precedence — operator may inject secrets without storing them in YAML |
| Config struct → log output | Secret fields MUST NOT appear in logs |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-03-01 | Information Disclosure | Logging the loaded `*Config` would leak `Postgres.DSN`, `Telegram.BotToken`, `SMTP.Password`, `HTTP.SessionSecret` | high | mitigate | This plan does NOT call any logger on the Config. Downstream plans (04, 09, 10) must redact secret fields before logging — checker rule: `grep -E "slog\\..*cfg\\.|slog\\..*Config\\." internal/<plan>/*.go` should not directly log a *Config. We mitigate further in plan 12 (serve) by adding a `Redact()` method if logging is needed. |
| T-01-03-02 | Tampering | `config.yaml` writable by non-root user → injection of malicious DSN | high | mitigate | systemd unit (plan 01) runs as `mitigador` user; operator-facing README in plan 12 will document `chmod 640 /etc/mitigador/config.yaml; chown root:mitigador`. Documented as deploy contract; out-of-scope for runtime enforcement in P1. |
| T-01-03-03 | Spoofing | Env var injection from a less-privileged sibling process could override config secretly | medium | accept | systemd `EnvironmentFile=-/etc/mitigador/mitigador.env` (already in plan 01 unit) scopes env vars to the service. Process isolation handled by the OS. |
| T-01-03-04 | Denial of Service | Malformed YAML triggers panic in viper unmarshal | medium | mitigate | `Load` wraps every viper error with `%w` and returns it; no panics. Tests cover nonexistent file, missing required field, malformed enum. |
| T-01-03-05 | Information Disclosure | Validation error messages echo field VALUES, possibly including secrets if validator surfaces them | high | mitigate | `Validate()` uses `fe.Value()` which returns the field value — for `Telegram.BotToken` or `SMTP.Password` this would leak the secret in the error message. **MITIGATION**: In Task 2, modify `Validate()` to redact value for any field named `Token`, `Password`, `Secret`, or `SessionSecret` — replace value with `<redacted>` before formatting. Verify with a unit test that error on missing `SessionSecret` does NOT echo any actual secret. |
