## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, table cell icon spacing |
| sm | 8px | Compact control spacing, list item inner padding |
| md | 16px | Default element spacing, card inner padding, form field gaps |
| lg | 24px | Section padding, card-to-card gaps, sidebar item spacing |
| xl | 32px | Layout section breaks, panel headers |
| 2xl | 48px | Major section separators |
| 3xl | 64px | Page-level vertical rhythm (top/bottom page padding) |

Exceptions:
- Touch target minimum: 44px height for all clickable elements (mobile-first: ISP operator acts from phone)
- Sidebar width: 240px (fixed, not on the 8-point scale — standard ops dashboard convention)

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Table rows, descriptions, form labels, alert detail text |
| Label | 12px | 600 (semibold) | 1.3 | Metric labels, badge text, column headers, status chips |
| Heading | 20px | 600 (semibold) | 1.2 | Page titles, panel/card headings, section names |
| Display | 28px | 600 (semibold) | 1.1 | Attack count badge on header, primary metric callouts |

Notes:
- Exactly 2 weights used: regular (400) and semibold (600).
- Mono font (system-ui monospace) used for IP addresses and PPS/BPS numbers to prevent layout shift as values update live.
- Alert messages (Telegram + email) always render in pt-BR regardless of the UI toggle. The pt-BR/en-US toggle (DASH-09) applies to the dashboard UI only — not backend-generated alert text.

## Color

Naive UI dark theme is the default for this ops dashboard (network operators work in dark environments; screen glare on a NOC floor). Light theme toggle is Claude's discretion for Phase 3.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#101014` (Naive UI dark `bodyColor`) | Page background, main content area surfaces |
| Secondary (30%) | `#1c1c21` (Naive UI dark `cardColor`) | Cards, sidebar, nav panel, table background, modal surface |
| Accent (10%) | `#18a058` (Naive UI `primaryColor` — green) | Primary action buttons, active nav item indicator, SSE "connected" status dot, active attack count badge border |
| Warning | `#f0a020` (Naive UI `warningColor` — amber) | Attack severity "medium", Telegram rate-limit queue warning, exporter "stale" status |
| Destructive | `#d03050` (Naive UI `errorColor` — red) | Attack severity "high", exporter "offline" status dot, error state headings |

Accent (green) reserved for:
1. The single primary CTA per screen (e.g., "Entrar" login button)
2. Active nav item left-border indicator
3. SSE stream connected-status indicator dot
4. "Active attacks" counter badge border when count > 0

Accent is NOT used for: table rows, general text, icons, disabled states, informational chips.

Semantic color usage:
- Green (`#18a058`): nominal / connected / normal operations
- Amber (`#f0a020`): degraded / warning / stale (exporter not seen for > 5 min)
- Red (`#d03050`): attack active / error / exporter offline

## Views and Interaction Contract

### View Inventory (Phase 1)

| View | Route | Auth Required | Source Requirement |
|------|-------|---------------|-------------------|
| Login | `/login` | No | DASH-01 |
| Dashboard (live attacks) | `/` | Yes | DASH-02 |
| Exporter Health | `/exporters` | Yes | DASH-05 |
| BGP Sessions (stub) | `/bgp` | Yes | DASH-04 (D-18) |
| Incident List | `/incidents` | Yes | PERS-01 |
| Incident Detail | `/incidents/:id` | Yes | PERS-01 + ALER-05 |

### View: Login (`/login`)

**Layout:** Centered card on page background. Card width: 360px. Card padding: 32px.

**Elements:**
- Heading: "Mitigador" (28px semibold)
- Sub-label: "Acesso operacional" (14px regular, secondary text color)
- Username field: `n-input` with label "Usuário"
- Password field: `n-input` type password with label "Senha"
- Primary CTA button: "Entrar" (full-width, accent green, semibold)
- Error state: `n-alert` type="error" below the form (see Copywriting)

**Interactions:**
- Enter key submits form
- Button shows loading spinner during POST `/api/auth/login`
- On success: redirect to `/`
- On failure: show error alert, do NOT clear password field

**Empty / error states:**
- Invalid credentials: see Copywriting section

### View: Dashboard — Live Attacks (`/`)

**Layout:** Two-column. Left: sidebar nav (240px fixed). Right: main content area.

**Main content structure (top → bottom):**
1. Header bar: page title "Ataques em Tempo Real" + SSE status indicator dot (green = connected, amber = reconnecting, red = disconnected) + locale toggle (pt-BR / EN)
2. Summary strip: 3 metric cards inline — "Ataques Ativos" (display 28px), "Exporters Online", "Alertas Hoje"
3. Attack table: live-updating via SSE, rows sortable by start time DESC

**Attack table columns:**
| Column | Type | Width |
|--------|------|-------|
| IP Alvo | monospace 14px | 140px |
| Vetor | badge chip | 120px |
| PPS | monospace 14px, right-aligned | 100px |
| BPS | monospace 14px, right-aligned | 110px |
| Duração | 14px | 90px |
| Score | mini progress bar (0–100) | 80px |
| Iniciado | relative time (e.g., "há 3 min") | 110px |

Attack row color coding:
- Score ≥ 80: row left-border red 3px
- Score 40–79: row left-border amber 3px
- Score < 40: no accent border

SSE behavior:
- On `attack:started` event: prepend row to table, flash row background for 1s
- On `attack:update` event: update PPS/BPS/score in-place, no flash
- On `attack:ended` event: strike through row, fade to 30% opacity, remove after 10s
- On SSE disconnect: show reconnecting indicator; table freezes in last known state (no spinner, no content wipe)
- On SSE reconnect: apply any queued events; do not duplicate rows

**Empty state (no attacks):** see Copywriting section.

### View: Exporter Health (`/exporters`)

**Layout:** Same sidebar + main. Table of configured exporters.

**Exporter table columns:**
| Column | Type |
|--------|------|
| IP do Exporter | monospace 14px |
| Tipo | badge (NetFlow v9 / IPFIX / sFlow v5) |
| Último Flow | relative time |
| Taxa de Chegada | monospace, flows/s |
| Override Sample Rate | value or "—" if none |
| Status | colored dot: Online / Stale / Offline |

Status logic:
- Online: last flow seen < 60s ago (green dot)
- Stale: last flow seen 60s–5 min ago (amber dot)
- Offline: last flow seen > 5 min ago, or never (red dot)

**Warming-up state:** If template warming is in progress (first 5 min after restart), show `n-alert` type="warning" above table: "Aguardando templates de flow de {N} exporter(s). Registros podem ser descartados até os templates chegarem."

**Empty state:** see Copywriting section.

### View: BGP Sessions Stub (`/bgp`)

**Layout:** Same sidebar + main. No table, only the empty state card.

This view is a permanent stub in Phase 1 per D-18. No BGP data is fetched. No loading state is shown.

Content: A single centered `n-empty` component with:
- Icon: network/server icon from `@vicons/ionicons5`
- Title: "Nenhuma sessão BGP configurada" (heading 20px)
- Description: "As sessões de mitigação BGP serão configuradas na Fase 2."
- No CTA button (nothing the operator can do in Phase 1)

### View: Incident List (`/incidents`)

**Layout:** Same sidebar + main. Static table (not SSE — historical data only).

**Incident table columns:**
| Column | Type |
|--------|------|
| ID | monospace 14px (ULID, truncated to 12 chars) |
| IP Alvo | monospace 14px |
| Vetor | badge chip |
| PPS Pico | monospace, right-aligned |
| BPS Pico | monospace, right-aligned |
| Score | numeric 14px |
| Início | absolute timestamp (DD/MM/YYYY HH:mm) |
| Duração | 14px |
| Status | badge: "Ativo" (green) / "Encerrado" (secondary) |

Row click navigates to `/incidents/:id`.

Pagination: 50 rows per page, `n-pagination` component.

**Empty state:** see Copywriting section.

### View: Incident Detail (`/incidents/:id`)

**Layout:** Same sidebar + main. Single-column detail panel.

**Sections (top to bottom):**
1. Breadcrumb: "Incidentes / {ULID}"
2. Header: IP alvo (display 28px monospace) + vector badge + status badge
3. Metrics row: PPS pico, BPS pico, score de confiança, duração total
4. Timeline section: `n-timeline` with `started`, optional `update` entries, `ended`
5. Raw detail: vetor, classificação, critérios (pps/bps/duration que trigaram)

## Interaction States Required

Every data-fetching component MUST implement all four states:

| State | Implementation |
|-------|---------------|
| Loading | `n-spin` centered, minimum 200ms display (prevents flash) |
| Success | Render content |
| Empty | `n-empty` with heading + body from Copywriting table above |
| Error | `n-alert` type="error" with message + retry button labeled "Tentar novamente" / "Retry" |

SSE-specific states (dashboard live attacks view):

| State | Visual |
|-------|--------|
| Connected | Green dot (8px, accent color) with no label |
| Reconnecting | Amber dot with label "Reconectando…" |
| Disconnected | Red dot with label "Conexão perdida. Tentando reconectar." |
| Connected after gap | Snapshot refresh: re-fetch `/api/attacks/active` once to sync state |

## Responsive Behavior

Primary audience acts from mobile (ISP operator on Telegram → opens dashboard link on phone):

| Breakpoint | Sidebar | Content |
|------------|---------|---------|
| ≥ 1024px (desktop) | 240px fixed, always visible | Full table layout |
| 640–1023px (tablet) | Collapsible, hamburger icon | Table scrolls horizontally |
| < 640px (mobile) | Hidden, bottom nav strip with icon-only tabs | Cards instead of table rows for attacks |

Touch target minimum: 44px height on all interactive elements (buttons, nav items, pagination controls).

Mobile card layout for active attacks (< 640px):
- IP + vector badge on line 1
- PPS / BPS on line 2 (monospace)
- Duration + score on line 3
- Row border-left: same severity color coding as desktop
