## New API Surface

### TopEntry struct
```go
type TopEntry struct {
    IP            netip.Addr
    Bps           uint64  // bytes/sec summed over 60s window
    Pps           uint64  // packets/sec summed over 60s window
    DominantProto string  // "udp" | "icmp" | "other"
}
```

### Store.Top method
```go
func (s *Store) Top(now int64, n int) []TopEntry
```
- Returns up to `n` entries sorted by `Bps` descending
- Secondary sort: IP string ascending (deterministic tie-break)
- Excludes: hosts with `Bps == 0`, hosts with `LastSec < now - WindowSize`
- `n <= 0` returns `[]TopEntry{}` immediately
- Empty store returns `[]TopEntry{}` (not nil)

## Concurrency Invariants

`Top` follows the exact same per-shard mutex pattern as `ActiveHosts`:
- Lock each shard individually
- Iterate `sh.hosts` under the lock
- Unlock before moving to next shard
- No cross-shard lock held at the same time → no deadlock risk

The direct ring walk (`hr.Buckets[i]` inside the shard lock) reads values that `Update` also writes under the same lock — no data races.

## Decisions Made

- **Direct ring walk over Snapshot:** `Top` iterates `hr.Buckets[i]` directly (O(60) ints per host) rather than calling `Snapshot` which allocates a `[]Bucket` slice per host. For an ISP with thousands of active hosts this avoids significant GC pressure.
- **Tie-break udp > icmp > other:** Deterministic and UDP-biased; volumetric DDoS is predominantly UDP amplification so the most operationally meaningful proto wins ties.
- **n as parameter:** Caller (Plan 02's HTTP handler) supplies `n=20`; tests can use small values. No magic constant in the library.
