## Why uPlot Bands for Stacked Area

uPlot does not have a first-class "stacked area" mode. The canonical approach (from uPlot's own demos) is:

1. **Cumulative sums**: feed three Y series where each is the running total up to that protocol layer.
   ```
   y1 = bps_udp
   y2 = bps_udp + bps_icmp
   y3 = bps_udp + bps_icmp + bps_other  (= total)
   ```
2. **`bands` config**: `{ series: [aboveIdx, belowIdx] }` tells uPlot to fill the canvas region *between* two series lines. Because the series are cumulative, each band represents exactly one protocol's contribution.

This is superior to the alternative (setting `fill` to each series' individual value and relying on draw order) because:
- Each band's area is precisely the protocol's contribution, not an overlap artifact
- No z-order fighting between fill areas
- Works correctly even when one protocol temporarily dominates (fills the full height naturally)

## Cleanup Invariants

| Hook             | Action                                    | Why |
|------------------|-------------------------------------------|-----|
| `onBeforeUnmount`| `ro.unobserve(containerRef.value); ro=null` | Disconnect ResizeObserver before DOM is removed |
| `onUnmounted`    | `chart.destroy(); chart=null`             | uPlot registers `window` resize/scroll listeners; `destroy()` removes them all |

The split between `onBeforeUnmount` (ResizeObserver) and `onUnmounted` (chart destroy) is intentional: `onBeforeUnmount` runs while the DOM is still attached, making `containerRef.value` valid for `unobserve()`. `onUnmounted` is the right place for `chart.destroy()` since uPlot owns its own DOM subtree inside `containerRef`.

## Data Transform: X Axis

`HostBucket.offset_s` is `0` for "now" and `59` for "59 seconds ago". uPlot expects X values in ascending order (left→right = old→new). We reverse the buckets array and negate the offset:

```
X[i] = -(props.buckets[n-1-i].offset_s)   →   [-59, -58, ..., -1, 0]
```

The result: the rightmost point is `0` (now), and older data trails off to `-59s` on the left — the most intuitive read for an operator watching live traffic.
